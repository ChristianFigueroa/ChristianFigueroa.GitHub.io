What is this?

JS Addons is just a collection of added functions and shortcuts that you can use while scripting. You can select which addons you want to include in your downloadable file.

Documentation

NodeList.prototype.forEach • HTMLCollection.prototype.forEach

Type: Function
Description: Similar to Array.prototype.forEach. Allows you to loop over a list of elements without converting them to a legitimate array.
Usage: Use exactly the same as Array.prototype.forEach.

1document.querySelectorAll("div").forEach(function (currElement, index, array) {currElement.doStuff()});
NodeList.prototype.loop • HTMLCollection.prototype.loop

Type: Function
Description: Loops over a list of elements and runs a function, with the function's this value equal to the current element in the list. This addon is made to return the first value returned from the given function. Basically, the function is supposed to test the element against a certain criteria. If that condition is met, the function should return a value that will be returned to the .loop() call. See below for an example.
Usage: Accepts two arguments. The first argument is required and should be the function to be run. The second optional argument is an array of arguments to be passed to the function when run. Two arguments are concatenated to the array, the index of the current element and the entire array.

1document.querySelectorAll("div").loop(function (givenArgument, index, entireArray) {
2 if (this.id == "arbitraryID") return index;
3}, ["givenArgument"]);

If [div#randomID, div#anotherID, div#arbitraryID, div#lastID] were the provided list of elements, the top function call would return 2. The first function call would test "randomID" against "arbitraryID", resolving to false and not returning anything. The next element would then be tested and again not return anything. The third loop would resolve to true and return index, which since the index of div#arbitraryID in the list is 2, would return 2. The fourth call would not happen since a value had already been returned.

Node.prototype.interval

Type: Function
Description: Allows for linear animation. A function is run in a setInterval loop whose this value is set to the node being called upon. You can pass in an additional numeric argument to determine how many times the interval function should be run before being cleared. Each time the function is called, the index argument passed to the function increases. You can use this to change the style of the element according to how far the interval is from being cleared. See below for an example.
Usage: Accepts five arguments, the first being required, the last four being optional. Argument one should be the function to be run in each call. The second argument is how many milliseconds should be between each function call. Default is 100ms. The third argument is how many times the function should run before being cleared. Set as null or undefined to run forever. Argument 4 should be an array of arguments to be passed to the function, or null/undefined. The fifth and final argument should be a callback function to be run after the interval is cleared. The function to be run in each interval will recieve any arguments defined in argument four, as well as two more arguments: the current iteration (zero-based numbering) and a function that, when called, will clear the interval early (this allows you to end an interval that would normally run indefinitely, or end an interval earlier than intended).

1document.getElementById("arbitraryID").interval(function (index, endEarly) {
2 this.style.width = index + "%";
3 if (index == 100) endEarly();
4}, 10, null, []);

In the above example, the function is run 100 times at 10ms between each call. After index == 100, endEarly is called, ending the interval. Each time the function is called, index increases by 1, making #arbitraryID increase its width by 1% each call. Since the function is run so quickly, it appears like #arbitraryID is growing in width in a linear fashion. This makes it very easy to animate the growth of #arbitraryID's width.

Node.prototype.ael

Type: Function
Description: Acronym for AddEventListener. Works exactly the same except an array of strings is provided instead of a single string. The array will be looped through and an event handler will be added to each string. Great for adding a function for touch and mouse events.
Usage: Use exactly the same as Node.prototype.addEventListener, except replace the string in argument one with an array of strings.

1document.getElementById("arbitraryID").ael(["mousedown", "touchstart"], function (event) {event.doStuff()});

The above is exactly the same as applying the function to both "mousedown" and "touchstart" events. No matter if the user is on a touch device or mouse device, the same handler will still be run.

Element.prototype.rect

Type: Alias Shortcut
Description: Exactly the same as calling .getBoundingClientRect() on an Element. Doesn't need the parentheses.
Usage: Use exactly the same as Element.prototype.getBoundingClientRect without the parentheses.

1document.getElementById("arbitraryID").rect;

The top would return the same as document.getElementById("arbitraryID").getBoundingClientRect();

String.prototype.escape

Type: Function
Description: Converts HTML characters into HTML entities, making the string safe to use as an element's innerHTML.
Usage: Call .escape() on any string to make it safe to use for innerHTML.

1document.getElementById("arbitraryID").innerHTML = "<span style="color:red;">TEST</span>".escape();

#arbitraryID's innerHTML would display as '<span style="color:red;">TEST</span>' instead of as 'TEST'.

String.prototype.unescape

Type: Function
Description: Converts HTML entities into HTML characters, making an escaped string valid HTML, the opposite of .escape.
Usage: Call .unescape() on any string to make it interpretable as HTML.

1document.getElementById("arbitraryID").innerHTML = "&lt;span style="color:red;"&gt;TEST&lt;/span&gt;".unescape();

#arbitraryID's innerHTML would display as 'TEST' instead of as '<span style="color:red;">TEST</span>'.

window.$ (or $_)

Type: Alias Shortcut
Description: Shortcut for document.querySelectorAll. If $ is already a declared variable (e.g. $ as an alias for jQuery), $_ should be used instead.
Usage: Call $ (or $_) with a stringified CSS selector to return all the elements matching that selector.

1$("div.class");

The top would return a NodeList of all the <div> elements whose class includes "class", exactly the same results as document.querySelectorAll("div.class");.

window.$$<

Type: Alias Shortcut
Description: Shortcut for document.querySelector. Note that this will only return the first element matching the selector, not a list of elements. Use $ instead for a list.
Usage: Call $$ with a stringified CSS selector to return the first element matching that selector.

1$$("div.class");

The top would return the first <div> element whose class includes "class", exactly the same result as document.querySelector("div.class");.

window.$id

Type: Alias Shortcut
Description: Shortcut for document.getElementById.
Usage: Call $id with the id of an element to return that element.

1$id("arbitraryID");

The top would return the element with the id "arbitraryID", exactly the same result as document.getElementById("arbitraryID");.

window.parseURLArgs

Type: Function
Description: Converts the query string of the URL into a JSON object.
Usage: Accept two arguments: the first determines whether two values for the same key should overwrite the first value, or be kept in an array. The second determines whether the values should be kept in a stringified form, or be evaluated to a different data type*. See below for an example. Both are false by default. Values should be encoded (using encodeURIComponent) to prevent characters like '&' or '=' from being interpreted incorrectly.

1// Example URL - https://example.com/thing?object={key:value}&number=3.1415&string=example1&array=[0,1,2]&string="example2"
2
3parseURLArgs();
4// Returns the following object
5{
6 object: "{key:value}",
7 number: "3.1415",
8 string: ""example2"",
9 array: "[0,1,2]"
10}
11
12parseURLArgs(true);
13// Returns the following object
14{
15 object: ["{key:value}"],
16 number: ["3.1415"],
17 string: ["example1", ""example2""],
18 array: ["[0,1,2]"]
19}
20
21parseURLArgs(false, true);
22// Returns the following object
23{
24 object: {
25 key: "value"
26 },
27 number: 3.1415,
28 string: "example2",
29 array: [0, 1, 2]
30}
31
32parseURLArgs(true, true);
33// Returns the following object
34{
35 object: [{
36 key: "value"
37 }],
38 number: [3.1415],
39 string: ["example1", "example2"],
40 array: [[0, 1, 2]]
41}

In the first object, the string key is set to ""example2"" because the URL has two values for 'string'. The last one is set to ""example2"", so it overwrites the original "example1". All values are kept in their exact stringified form (hence the surrounding quotes around ""example2"").

In the second object, every value is kept as an array. With the 'string' key, instead of ""example2"" overwriting "example1", it is added to the end of the array so that every value will appear in the overall object.

In the third object, each value is interpreted literally. The first value is interpreted as a JSON object. The second is interpreted as a number. The third remains a string (it's surrounding quotes were parsed to tell the interpreter to parse it as a string). The fourth is interpreted as an array.

The fourth object is a combination of the second and third objects. Each value is added to an array and interpreted literally.

*Note that in order for strings to be parsed literally (second argument = true), String.parse must be included in the file.

String.parse

Type: Function
Description: Very similar to eval, but a little smarter. Allows you to turn a stringified value into a JavaScript data type (this addon is required for window.parseURLArgs's second argument).
Usage: Call String.parse on any string to interpret it as a JavaScript data type. If the value could not be interpreted as any data, it will return in its stringified form (e.g '{key:value}' would be interpreted as an object, but 'randomString' is nothing but a string).

1String.parse("just a string");
2// Would return "just a string"
3String.parse("'[quoted,string,that,looks,like,array]'");
4// Even though it would normally be interpreted as an array, the surrounding quotes inside the string force the interpreter to return the value as a string. "[quoted,string,that,looks,like,array]"
5String.parse("[actual, 'array']");
6// Returns an actual array since there are no surrounding quotes. Each element in the array is parsed individually. Returns ["actual", "array"]
7String.parse("[-Infinity, true, /regex/gmi, {key:value}, undefined]");
8// Returns [-Infinity, true, /regex/gmi, {key: "value"}, undefined]
9String.parse("{objects: work, like arrays: [and, {can:'be'}, inter-nested], inside: each other}");
10// Returns {objects: "work", "like arrays": ["and", {can: "be"}, "inter-nested"], inside: "each other"}

The top would return the first <div> element whose class includes "class", exactly the same result as document.querySelector("div.class");.

Date.prototype.toElapsedString

Type: Function
Description: Returns a string describing the amount of time that has passed (or is yet to pass) between the Date object and the current time.
Usage: Accepts one argument: a boolean indicating whether to use the long form of a word ('seconds', 'minutes', 'hours') or the shortened version ('s', 'm', 'h').

1new Date("January 1, 2000").toElapsedString();
2new Date("January 1, 3000").toElapsedString(true);
3new Date().toElapsedString();

The first call would be formatted as "x years ago" where x is the amount of years since January 1, 2000. The second would be formatted as "xy from now" where x is the amount of years from now until January 1, 3000.. The third and final call would return "now" since no time is passing between the Date and the function call.
