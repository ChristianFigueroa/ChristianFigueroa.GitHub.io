These three examples are all different problems from coursework while studying in my undergraduate career. Since they're just homework problems, there isn't as much room to go quite as above-and-beyond as I'd like (the way this internship would allow for!), but nonetheless, I still love stylizing mathematics, even to more than is what required of me from the class. (E.g., I submitted every problem set of my multivariable calculus course in a LaTeX PDF, even though it took at least three hours longer each time than just drawing it, simply for that feeling of appreciating how beautiful my homework looked by typesetting it at the end.)

Example #1

            \draw[smooth,variable=\x,domain=-5:5,line width=1pt,draw=red!\the\numexpr\i*10/3+50\relax!blue]

        }

        \draw[line width=1pt,draw=red!50!blue] (-5,-5) edge (5,5)
            (-5,5) edge (5,-5);

        \foreach \i in {3,6,...,15} {
            \draw[smooth,variable=\y,domain=-5:5,line width=1pt,draw=red!\the\numexpr\i*-10/3+50\relax!blue]
                plot (sqrt{((\y)^2+\i)}, \y);
            \draw[smooth,variable=\y,domain=-5:5,line width=1pt,draw=red!\the\numexpr\i*-10/3+50\relax!blue]
                plot (-sqrt{((\y)^2+\i)}, \y);
        }
    \end{scope}

    \begin{scope}[xshift=4.5cm]
        \draw[shading=axis,bottom color=red,top color=blue] (0,-4) rectangle (.5,4);
        \foreach \i in {-15,-12,...,15} {
            \draw (0,\i*4/15) -- (-.2,\i*4/15) node [left] {$\i$};
        }
    \end{scope}
\end{tikzpicture}

This is a colorized contour map of the 3D function $z=y^2+x^2$. This was a subproblem from a problem set at the start of my multivariable calculus course. It uses tikz in LaTeX to draw the graph. To represent it on a 2D plane, I used a color gradient with red values being lower and blue values being higher along the (hidden) z-axis.

Example #2

# Generate 40 random samples from a beta distribution
raw_samples <- replicate(40, rbeta(65, 10, 1))
# Get their means and standard deviations
ci_samples <- raw_samples %>% {data.frame(
    means=apply(., 2, mean),
    sds=apply(., 2, sd)
)}

# Calculate standard errors and lower/upper ranges for a 95% confidence interval
ci_samples$ses <- qt(0.975, 65 - 1) * ci_samples$sd / sqrt(65)
ci_samples$lower <- ci_samples$mean - ci_samples$se
ci_samples$upper <- ci_samples$mean + ci_samples$se

# Actual mean of a beta distribution
true_mean = 10 / (10 + 1)
# Form a new data frame that we will use to draw the lines
ci_lines <- ci_samples %>% {data.frame(
    y=rep(1:40, 2),
    x=c(.$lower, .$upper),
    # Keeps track of whether this CI intersects the true mean
    intersecting=.$lower < true_mean & .$upper > true_mean
)}

ggplot() +
# Draw the lines first
geom_line(aes(
    x=ci_lines$x,
    y=ci_lines$y,
    group=ci_lines$y,
    color=ci_lines$intersecting
)) +
# Highlight the ones that don't intersect
scale_color_manual(values=c("red", "#777777")) +
# Draw the lower range of the CI
geom_point(aes(
    x=ci_samples$lower,
    y=as.numeric(row.names(ci_samples))
)) +
# Draw the upper range
geom_point(aes(
    x=ci_samples$upper,
    y=as.numeric(row.names(ci_samples))
)) +
# Add a line for where the true mean is
geom_vline(xintercept=true_mean) +
# Change axes and legend
xlab("95% Confidence Intervals") +
theme(
    legend.position="none",
    axis.text.y=element_blank(),
    axis.ticks.y=element_blank(),
    axis.title.y=element_blank()
)

This diagram shows an intuitive visual for a 95% confidence interval. The black vertical line shows the true mean and 40 random samples were taken to show which of them contained the true mean in their 95% confidence interval. 38/40 (95%) happened to intersect, perfectly illustrating the meaning of the 95%. Since this was for a timed assignment in my biostatistics course, I wasn't able to make it as stylistically appealing as I would have today, but with more time and maybe an ability to animate, this visual could be made to look even better.
